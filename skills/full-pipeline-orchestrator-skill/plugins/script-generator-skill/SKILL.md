# 短剧剧本生成技能

## 技能说明

将用户的一句话创意扩写为结构完整、节奏紧凑的短剧剧本。支持两种剧本来源：
1. 用户输入一句话创意：调用本技能生成完整剧本
2. 用户上传现有剧本/直接提供完整剧本文字：跳过本技能的生成逻辑，直接读取内容写入flow.json

公共规则见主 `SKILL.md` 与 `FLOW_CONTRACT.md`；本插件只定义剧本内容目标与输出格式。

## 输入要求
【一句话生成场景】：
- 用户创意
- 视觉风格
- 目标媒介
- 集数标识：如 `ep-01`
【上传剧本场景】：
- 自动读取项目根目录`/temp`下的剧本文件，无需额外输入

## 执行流程

【前置分支判断】
- 若为用户明确使用上传剧本场景：
  1. 读取项目根目录`/temp`目录下最新修改的`.md`/`.txt`剧本文件
  2. 校验文件有效性：存在、非空、格式支持，校验失败返回标准化错误提示
  3. 读取完整剧本内容，提取标题，无需生成直接返回
- 若为用户输入一句话创意生成场景：
  1. 提取创意中的核心冲突、人物关系、情感线索
  2. 构建短剧结构：开场、铺垫、发展、高潮、结局
  3. 写出完整剧本，包含：
     - 故事大纲
     - 人物设定
     - 场景列表
     - 完整剧情

## 输出要求
- 【通用规则】
  - 正式文件由主 Agent 落盘到 **`ep-xx/script.md`**（相对于 `canvas_project_dir`，见 `FLOW_CONTRACT.md`）
  - 主 Agent 必须遵守 **flow-first** 规则：先写入 `flow.json` 剧本字段，校验通过后再导出磁盘 `script.md`
  - 返回两个标题字段：
    - 短标题：简洁剧集展示标题（用于`episodes[].title`字段）
    - 完整剧本标题：详细剧本标题（用于`episodes[].script.title`字段）
  - `episodes[].script.content` 与文件正文须全文一致
- 【上传剧本场景特殊规则】
  - 完全保留用户上传剧本的原有格式、内容、排版，不强制改写为生成剧本的模板结构
  - 剧本标题优先取上传文件的文件名（不含后缀），内容和原文件完全一致

### 输出文件格式

```markdown
# ep-01 完整剧本

## 故事大纲
[100-200 字概述]

## 人物设定
| 角色名 | 年龄 | 性格 | 外貌简述 |
|--------|------|------|---------|
| ...    | ...  | ...  | ...     |

## 场景列表
| 场景编号 | 场景名 | 简述 |
|---------|--------|------|
| S01     | ...    | ...  |

## 完整剧情

### S01 - <场景名>
**场景描述**：[环境、时间、氛围]

**角色名**：（动作/表情）"台词内容"
```

## 质量标准
【一句话生成场景】：
- 起承转合清晰
- 冲突明确、节奏紧凑
- 人物与台词有辨识度
- 场景数量合理，无明显逻辑漏洞
【上传剧本场景】：
- 仅校验内容非空有效，不限制格式、结构、篇幅，完全保留用户原作内容
