# 导演分析技能

## 技能说明

将剧本转化为可执行的视觉叙事蓝图：拆剧情点、讲戏、提人物清单、提场景清单。

公共规则见主 `SKILL.md` 与 `FLOW_CONTRACT.md`（导演产出写入 **`sceneNode.data.scenes`**）；本插件只定义导演分析的内容结构与剧情点命名。

## 输入要求

- 剧本内容：`ep-01/script.md`
- 集数标识：如 `ep-01`
- 视觉风格
- 目标媒介

## 执行流程

1. 通读剧本，梳理叙事线、人物关系、情感走向
2. 拆成若干剧情点：`P01`、`P02`...
3. 为每个剧情点给出建议时长（**单镜 1～15 秒**，见下「单镜时长约束」与主 `SKILL.md` **[视频管线模式]**）
4. 为每个剧情点“讲戏”，覆盖：
   - 画面内容
   - 人物动作
   - 台词 / 声音
   - 镜头感觉
   - 光影氛围
5. 提取人物清单与场景清单
6. 若全局服化道文件已存在，则给每项标注 `新增 / 复用 / 变体`

## 单镜时长约束（P0）

- 统一使用multi_reference模式（Seedance 2.0），每个剧情点建议时长 **不得超过 15 秒**（与 2.0 常见单次成片上限对齐）。
- 若一段叙事或镜头逻辑 **超过15秒上限**，必须 **拆成多个** `Pxx`（多个镜），不得用一条超长镜硬塞。
- 分镜阶段「建议成片时长」、七段式与 `videoPrompt` 须与此一致，避免请求参数超出 API 上限。

## 与画布分镜的关系

- 剧情点 `P01`、`P02`... 按顺序对应 `s1` / `S-1`、`s2` / `S-2`...
- 主 Agent 在导演阶段完成后执行“导演拆镜回写”
- 初版镜头列表写入 `sceneNode.data.scenes`；详细 `imagePrompt` / `videoPrompt` 保留到后续镜级磁盘文档阶段
- `sceneNode.data.scenes[].content` 必须保持叙事向

### 输出文件格式

```markdown
# ep-01 导演分析

## 剧情拆解
### P01（8s）
**对应剧本**：（建议）剧本中场景 / 场次名称或段落位置 + 本镜关键对白摘要 1～2 句，便于分镜【台词】与原文对齐  
（括号内时长须 ≤ 15秒单镜上限；若需更长叙事请拆成 P02、P03…）  
[导演阐述内容，3-5 句话]

### P02（10s）
**对应剧本**：（同上，可写「无对白」若本镜纯动作 / 环境）  
[导演阐述内容]

## 人物清单
| 人物名 | 年龄 | 外观关键词 | 素材状态 |
|--------|------|-----------|---------|
| 张三   | 25   | 高个子，短发，穿蓝色牛仔外套 | 新增 |

## 场景清单
| 场景名 | 时间 | 光线/色调 | 氛围关键词 | 素材状态 |
|--------|------|----------|----------|---------|
| 张三的家 | 夜晚 | 暖黄台灯 | 温馨，怀旧 | 复用 |

## 导演阐述
[整体视觉风格、叙事节奏、色彩倾向]
```

## 输出落盘

主 Agent 将上述内容写入 **`ep-xx/director-analysis.md`**（相对于 `canvas_project_dir`，与 `FLOW_CONTRACT.md` 一致）。

## 质量标准

- 每个剧情点建议时长在 **1～15 秒** 内；超过须拆镜
- 每个剧情点至少 3-5 句话
- 建议每个 `Pxx` 下包含 **对应剧本** 一行，指向 `ep-xx/script.md` 中场次或对白，减轻分镜阶段对齐成本（非强制，但强烈推荐）
- 讲戏足够具体，分镜师可直接提取动作、运镜、光影、台词
- 人物与场景清单可直接供服化道设计使用
